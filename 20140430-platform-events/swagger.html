The Swagger™ Affair

One of the great features of the DSP is the Live API/SDK. This is an application that is provided through the DSP admin application. At startup, it looks for any Swagger™ configuration files in the known resource, model and application paths. For all system-provided resources and services, these files end in .swagger.php. Applications and user-defined services can have Swagger™ configuration files in either PHP or Javascript.

These files are aggregated and presented in the Live API/SDK application, sorted by service. Once you expand a service, the operations provided are displayed. A user may then read the documentation and explore each API call. In addition, and this is cool part, the user can put all the arguments for any call into this application and the REST call will be made, and the results displayed.

Clicking the above option opens and loads the operations available by service. Clicking a service will expand and display the operations available. For instance, figure 2 shows the new goodies that lie beneath the system service.
